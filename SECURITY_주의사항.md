# 보안 및 매매 주의사항

## API 키 관리

- `OPEN_DART_API_KEY`, `NAVER_CLIENT_ID`, `NAVER_CLIENT_SECRET`은 GitHub Secrets에만 넣으세요.
- 노트북, README, CSV, YAML 파일에 API 키를 직접 적지 마세요.
- 저장소를 public으로 만들 경우 특히 주의하세요.

## 자동매매 관련

이번 패키지는 주문 실행 기능을 넣지 않았습니다.

현재 목적은 아래 흐름입니다.

```text
자동 리포트 생성
→ 사람이 확인
→ HTS/MTS에서 수동 매매
```

자동 주문으로 넘어가려면 별도 단계가 필요합니다.

1. 모의투자 API
2. 주문 전 확인 버튼
3. 1일 손실한도
4. 종목당 최대 비중
5. 중복 주문 방지
6. 장중 장애 대응
7. 실계좌 전환 전 최소 2~4주 검증

## 투자 유의

이 리포트는 투자 판단 보조용입니다.
매수·매도 권유가 아닙니다.
실제 매매 전 현재가, 호가, 공시, 뉴스, 거래정지/투자경고 여부를 반드시 직접 확인하세요.
